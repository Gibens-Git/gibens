BUGS FIXED
----------
1. Avatar not persisting after refresh
   - Root cause: upsertUser() used Supabase .upsert() which tries an INSERT
     first. Since full_name/role are NOT NULL, the insert failed silently
     and avatar_url was never saved to the DB.
   - Fix: changed .upsert() to .update().eq('id', id) in packages/supabase/src/queries.ts
   - File: packages/supabase/src/queries.ts

2. Customer profile pictures not showing in pro app Messages list
   - Root cause: get_vendor_chat_list RPC does not return other_avatar_url.
   - Fix: after loading threads, do a follow-up query to jobs→users to
     batch-fetch customer avatars and patch the thread list.
   - File: apps/pro/src/pages/Messages.tsx

3. AI support assistant not referring customers to relevant service categories
   - Root cause: system prompt described Gibens as "home services only" with
     no category list, so the AI said car keys were "out of scope" instead of
     directing the user to the Locksmith category.
   - Fix: rewrote system prompt to include all 30 categories with examples
     (e.g. lost car keys → Locksmith), added rule to always check category
     list before declining. Updated welcome message to signal broader scope.
   - File: apps/customer/src/pages/Support.tsx
