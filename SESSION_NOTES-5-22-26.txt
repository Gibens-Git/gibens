SUPABASE RLS POLICY CHANGE (run in SQL Editor)
================================================================================

The jobs SELECT policy originally required vendors to have their GPS location
set in vendor_profiles (ST_DWithin check). Since location is never populated
on signup, vendors saw zero jobs.

REPLACED with category-only match (run this if not already applied):

  DROP POLICY "jobs_customer_own" ON jobs;

  CREATE POLICY "jobs_customer_own" ON jobs FOR SELECT USING (
    customer_id = auth.uid() OR auth_role() = 'admin'
    OR (
      auth_role() = 'vendor'
      AND status IN ('open','bidding')
      AND EXISTS (
        SELECT 1 FROM vendor_profiles vp
        WHERE vp.user_id = auth.uid()
          AND vp.category = jobs.category
      )
    )
  );

ADMIN USER SETUP
================================================================================

Admin app has no registration page. To create/promote an admin user:

  -- Promote existing user to admin (by email):
  UPDATE users SET role = 'admin'
  WHERE id = (SELECT id FROM auth.users WHERE email = 'your@email.com');

  -- Or insert a new admin row after creating auth user in Supabase dashboard:
  INSERT INTO users (id, role, full_name)
  VALUES ('paste-uuid-here', 'admin', 'Admin');

NOTE: Once a user is promoted to admin, they can no longer post jobs
(jobs_customer_insert policy requires role = 'customer').
Use a separate customer account for testing the customer app.
